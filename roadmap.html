Roadmap

Where We’re Headed

Our roadmap reflects the product architecture, runtime governance, and optimization workflows we’re building toward across the iMorph platform.

2026 Q1

Agent Architecture Research and Core Modeling

1. Agent Flywheel Architecture: Completed the overall architecture definition for Agent Studio and Insights, establishing a closed-loop system across Build → Test → Monitor → Deploy → Suggestions.

2. Runtime Context Assembly Research: Researched and validated the dynamic assembly model for Journey, Guidelines, Reference, Actions, and Variables.

3. Agent Modeling Standardization: Established standards for minimal modeling units, including Journey, Branch, Edge, Guideline, Retriever, and Variable.

4. Runtime Governance Research: Completed research on runtime governance areas, including Runtime Snapshot, Deployment Governance, PII Masking, Handoff, and Fallback.

2026 Q2

MVP Productization and Core Workflow Launch

1. Agent Studio MVP: Launched the foundational workspace capabilities for KnowledgeHub, Build, Test, Deploy, Monitor, and Suggestions.

2. Build-time Runtime Orchestration: Supported structured orchestration of Main Journey, Branch, Edge, Actions, and Variables.

3. White-box Test Validation: Supported Test Group, batch regression testing, Evaluate Answer, and Reference / Guideline hit analysis.

4. Deploy Runtime Foundation: Supported Published Definition, Deployment Snapshot, Web Widget Runtime, and the foundational Gateway.

2026 Q3

Runtime Observability and Optimization Loop

1. Request-based Monitor Debugger: Supported runtime detail inspection for RequestTrace, Guideline Match, Tool Calls, Generation, Analysis, and Span Tree.

2. Runtime Explainability: Supported Runtime Assembly Explainability and white-box diagnostic capabilities.

3. Evidence-first Suggestions Optimization: Automatically generated optimization suggestions based on Conversations, Ratings, Monitor Trace, and Test Failures.

4. Validation-driven Iteration Loop: Connected the continuous optimization workflow of “Issue Discovery → Suggestion Generation → Patch Creation → Regression Validation.”

2026 Q4

Agent Operating System Platformization

1. Unified Runtime Governance: Supported governance capabilities including Rollout, Suspend, Resume, Rollback, Audience Routing, and Channel Governance.

2. Insights Intelligence Layer: Enhanced global experience analytics capabilities across Conversations, Topic Explorer, and CX Dashboard.

3. Cross-module Continuous Optimization: Established a unified optimization flywheel across Build, Test, Monitor, Deploy, and Suggestions.

4. Multi-agent Scalability: Supported lifecycle governance, asset reuse, and cross-business scenario expansion for multiple agents.

Release Notes

Product Updates by Version

Track the latest platform capabilities, runtime improvements, and workflow upgrades shipped across iMorph releases.

v0.6.3

Enhanced agent integration, deployment, and openness capabilities, enabling broader business automation and voice-based use cases.

  1. Tool Configuration: Build now supports HTTP tool creation and invocation, allowing agents to connect with external systems and automate business workflows.
  2. Voice Channel Deployment: Deploy now supports voice channels with automatic phone number provisioning and voice agent configuration, accelerating voice agent rollout.
  3. Chat OpenAPI: Introduced Chat OpenAPI, enabling developers to access agent conversation capabilities through standard APIs and integrate them into business systems and applications.

v0.6.2

Expanded orchestration, configuration, and evaluation capabilities to enable more sophisticated and scalable AI agent deployments.

  1. Flexible Agent Creation: Create agents from scratch or start with templates to balance flexibility and speed.
  2. Multi-Journey Support: Manage multiple workflows within a single agent, reducing maintenance effort while supporting more complex use cases.
  3. Enhanced Configuration: Support variable write-backs and advanced workflow controls for greater business flexibility and dynamic decision-making.
  4. Self-Play Evaluation: Automatically evaluate agent performance through simulated conversations, helping teams identify configuration issues and workflow risks before deployment.

v0.6.1

Enhanced agent testing, runtime performance, and platform governance to help teams identify issues faster and deliver more reliable AI experiences.

  1. Self-Play Simulation: Automatically generate conversation scripts to reduce manual testing effort and improve scenario coverage.
  2. Lite Mode: Run agents in Lite Mode to significantly reduce latency and deliver a smoother experience for voice and other real-time interactions.
  3. Built-in Counter Tool: Support complex state management and conditional logic, expanding agent capabilities in rule-driven workflows.
  4. Data Access Controls: Strengthened data isolation and security to support multi-team collaboration and enterprise governance.

Beta v0.5.0

Delivered the core capabilities required to create, test, monitor, and deploy AI agents, enabling business teams to build and validate agents faster and bring them into production with confidence.

  1. Login & Registration: Established a unified account system, making it easier for users to get started with building and managing agents.
  2. Agent Creation: Automatically generate onboarding flows to reduce setup effort and accelerate time to launch.
  3. Knowledge Management: Centralize and maintain knowledge to improve response accuracy and consistency.
  4. Preview Testing: Test conversations with agents in real time and view matched rules and execution paths, making it easier to validate configurations and optimize agent performance before deployment.
  5. Batch Testing: Validate agent performance across multiple scenarios before launch, reducing production risks.
  6. Conversation Monitoring: Access detailed conversation logs to quickly identify issues and continuously optimize agent performance.
  7. Web Deployment: Publish agents to web channels with one click, accelerating business adoption and validation.
